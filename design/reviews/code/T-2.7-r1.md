## Summary

T-2.7 is an M2 integration / wiring regression (of T-2.3/T-2.4): the production tool registry the
live `codingagent` run offers the model registered only the seven file/search/run tools and never
`spawn_subagent` (C13, ADR-0010), `read_memory`, or `write_memory` (C12, ADR-0007), and never
instantiated the `ChildAgentLoopFactory` production seam. The class that did the composition
(`AgentLoopFactory`) is JaCoCo-excluded, so no unit test caught the gap — the same live-vs-mocked
shape as the M0 D1/D2 defects.

The fix extracts the composition logic out of the coverage-excluded factory into a new
package-visible, gate-covered `ToolRegistryComposer`, which assembles all ten tools and the
sub-agent orchestrator over the production child-loop seam (`() -> childLoop.run(prompt)` over a
REAL nested `AgentLoop` reusing the parent's `ModelClient` — D2-safe by construction). The factory
narrows to the one un-coverable step (building the live `ModelClient` from real SigV4 credentials)
plus handing the composer its collaborators; the `create` signature gains a `SessionStore` (passed
from `Main`'s two already-present `SessionStore.forUserHome()` call sites) so the orchestrator can
open each child's OWN session log. `mvn clean verify` is green: 832 tests, 0 failures, JaCoCo
BUNDLE LINE gate (0.80) met (overall 92%); the new composer is 97% line-covered and the wiring is
now pinned by `LiveToolRegistryCompositionTest`.

### Nits

**[N1] [Readability] `ToolRegistryComposer.java` — `read_memory` constructed twice (parent + child registries)**
- Why: `parentRegistry()` and the private `childToolRegistry()` each `new ReadMemoryTool(memoryStore, repoKey)`.
  This is intentional — each `ToolRegistry` owns its own handler instances and `ReadMemoryTool` is a cheap
  stateless adapter over the shared store — but a reader sees two near-identical constructions. Clean Code
  tolerates this given the two registries are genuinely distinct collaborators (the child must NOT carry
  write_memory/spawn_subagent). Flagged for awareness, not change.
- Suggestion: Keep as-is. Extracting a `readMemoryTool()` helper would save one line at the cost of a
  level of indirection over a one-arg constructor. No action required.

### Praise

**[P1] The extracted seam makes the live wiring gate-covered without faking anything.**
- The composer takes the already-built `ModelClient` and injectable `MemoryStore`/`SessionStore`, so the
  test drives the *same* composition path production uses — over a scripted Bedrock double and temp-dir
  stores — rather than rebuilding a hand-rolled tool list. The sub-agent assertions exercise a real nested
  `AgentLoop` over the real wire path and check the D2 contract on the actual Converse re-call request
  (text member populated, json null), not field presence. This is exactly the live-reachability test the
  task asked for, and it would have caught the original gap.

## Design-doc alignment

- **ADR-0010 (sub-agent orchestration/isolation):** the child runs its OWN nested `AgentLoop` over the
  SAME `ModelClient` (reuse-by-construction D2 path), with its OWN session log (`SessionStore.openLog`,
  never the parent's), a fresh no-inherited-grants gate (`parentGrants.forSubAgent`), N=1 bound from
  `config.subAgentMax()`, 600s default cap. The child tool registry excludes `spawn_subagent` (no
  unbounded recursive spawning — the v1 N=1 / OOS stance). ✅
- **ADR-0007 (two-tier memory):** memory tools wired over `MemoryStore.forUserHome()` (`~/.codingagent`);
  read_memory = Class R, write_memory = Class X; write_memory carries the session EventLog (MEMORY_WRITE
  provenance, AC-12.4), boundary clock, origin session, repoKey. ✅
- **ADR-0005 (boundary-captured):** the child-session-id supplier and the clock are captured in
  `AgentLoopFactory` (the production boundary), not generated inside the orchestrator. ✅
- **INV-10/11/12 / AC-10.6 / AC-17.*:** preserved by reusing the unit-tested `SubAgentOrchestrator`
  (C13) unchanged — the composer only constructs it; the orchestrator's isolation/bound/lineage logic is
  the same code `SubAgentOrchestratorTest` pins.
