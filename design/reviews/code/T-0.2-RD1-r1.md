## Summary

Surgical regression fix for DEFECT D1: `ConfigDefaults.MODEL_ID` was the bare model id
`anthropic.claude-opus-4-8`, which on-demand Converse for Opus rejects with a 400
`ValidationException` ("... Retry your request with the ID or ARN of an inference profile ...").
The fix changes the compiled-in default to the ADR-0001-pinned cross-region inference-profile
form `us.anthropic.claude-opus-4-8`, updates the surrounding Javadoc (both `ConfigDefaults` and
`ResolvedConfig`) to explain WHY, re-pins the matching test oracles, adds a shape regression test
that would have caught the bare-id default, and aligns ADR-0001's pinned-default note with the
code. Scope held tight to the config default; no resolver logic, no unrelated defaults, no
test-local arbitrary model-id constants, and no fixtures were touched. Full `mvn clean verify`
green: 422 tests, 0 failures, JaCoCo gate met.

### Praise

**[P1] [Testing] `ConfigDefaultsTest.java:39` ŌĆö shape-based regression test, not a brittle literal twin**
- Why: The new `modelId_isInferenceProfileFormNotBareOnDemandId` asserts the *shape* the spec
  pins (region/scope-prefixed inference-profile id, NOT a bare `anthropic.*` on-demand id) rather
  than re-stating the exact literal. Verified the predicate catches both the original bare defect
  and a hypothetical future bare regression (`anthropic.claude-opus-4-9`) while accepting both
  ACTIVE inference-profile forms (`us.*`, `global.*`). It cites the real-Bedrock ValidationException
  as motivating evidence and traces the oracle to NFR-MODEL-DEFAULT / ADR-0001.
- Suggestion: none ŌĆö this is the right way to guard a regression that "silently passed" before.

### Discussion

**[D1] [Discussion] `ConfigResolverTest.java:330` ŌĆö fixture still carries the bare model id (intentional, flagged for awareness)**
- Why: `fixtures/config.global.yaml` declares `modelId: anthropic.claude-opus-4-8`, and
  `ct_sch_13_globalFixtureValidates` asserts that bare value. This is correct *for that test* ŌĆö
  its oracle is AC-8.1 (configurable values bind through), not NFR-MODEL-DEFAULT, so the fixture is
  an arbitrary user-supplied value, not the pinned default. Left unchanged deliberately. Same
  reasoning applies to the test-local `private static final String MODEL_ID = "anthropic.claude-opus-4-8"`
  constants in `ConverseWireMapperTest`, `ModelClientTest`, `OneShotRunnerTest`, and `AgentLoopTest`,
  and to the persistence/codec fixtures ŌĆö none of these is the config default.
- Suggestion: no action for this task. If a future reader wants fixtures to model a realistic
  default, that is a separate, non-load-bearing cosmetic change (and would be `Discussion`, never a
  defect). Surfaced only so the bare-id literal's continued presence in the tree is not mistaken for
  an incomplete fix.
