## Summary

T-1.1 replaces `Main`'s interactive stub with a real REPL (04-apis § 1.1/§ 1.4): a read-eval loop that reads a developer line and runs it as either an in-REPL slash-command or an agent-loop turn, streams the assistant's answer back (the completed `LoopOutcome` final text, written to stdout — the CLI owns user-facing output per 04-apis § 1.6), and repeats until the session ends. The work splits along the project's established seam: a **testable read-eval class** (`ReplRunner`, 100% line + 100% branch) plus a **production-only composition** in `Main` (JaCoCo-excluded, like `runOneShot`/`AgentLoopFactory`). New classes: `ReplRunner` (the loop, with injected seams — a `ReplLoop` turn seam, a `Supplier<String>` line source returning `null` at EOF, a `BooleanSupplier` interrupt flag, the permission mode, and out/err streams — so it is driven by a real `AgentLoop` over a scripted Bedrock double for the inline-approval test and by substitute loop seams for the branch tests, no live AWS) and `InteractiveApprover` (the REPL approval seam: presents `GateRequest.presentation()` — the exact operation — *before* reading the developer's y/N answer, AC-10.1; a non-affirmative answer incl. blank/unrecognized/EOF denies, fail-closed, AC-10.2). Slash-commands in T-1.1 scope: `/exit` → clean exit 0 (exit-code contract "interactive session exited cleanly"); `/mode` and `/permission` → show the current permission mode (04-apis § 1.4 "show ... current settings", US-8/9 — show-only, since the gate is built with a fixed mode at factory time; mid-session adjust is a documented later seam); any other `/command` → reported as unknown, never silently ignored (the task's § 1.4 scope note; `/compact`/`/attach`/`/remember`/`/model` map to M2/M4). EOF (Ctrl-D) ends the session cleanly (exit 0, G3). The **real SIGINT→130 path** the coordinator scoped in is wired: `Main` installs a `sun.misc.Signal("INT")` handler (the one non-portable, coverage-excluded production step) that sets the shared interrupt flag `ReplRunner` polls and interrupts the session thread to cancel the in-flight step; `ReplRunner` maps both the observed flag and a caught `InterruptedRunException` (the existing modelled-SIGINT seam OneShotRunner maps) to exit 130, reconciling the real handler with that seam (cli-exit-codes § 2 "130 always wins"; CT-EX-4/5; CT-SM-4). The session stays resumable across the interrupt because the event log flushes per event (NFR-LOG-DURABILITY, T-0.4). A failed *turn* does not kill the REPL (unlike one-shot): a surfaced edge reason, a model-backend/credential error, a blocking denial, or an unexpected fault is reported and the session continues (the developer chooses the next step, AC-10.2 spirit) — except a SIGINT (→ 130) and a fatal `PersistenceException` (→ exit 1, AC-13.4 — continuing would silently drop events). `mvn clean verify` is green: **450 tests** (30 new across `ReplRunnerTest` + `InteractiveApproverTest`; 4 stub-era `Main`-level tests removed/repointed — see [m1]), 0 failures/errors/skipped; `ReplRunner` and `InteractiveApprover` both 100% line + 100% branch; JaCoCo "All coverage checks have been met" (0.80 floor). One Minor ([m1], the interactive composition in `Main` is untested-by-design — it builds a live Bedrock client and reads stdin; its logic is tested at the `ReplRunner` layer; the composition is confirmed at the G1 real-Bedrock smoke test). One Discussion ([D1], the REPL drives each prompt as an independent `AgentLoop.run` turn — cross-turn model context continuation is C15/T-1.2's replay→messages[] job, not pinned by T-1.1's AC set; surfaced so it is an explicit, traceable seam, not a silent narrowing). No Blockers, no Majors.

### Minors

**[m1] [Testing] `cli/Main.runInteractive` / `installSigintHandler` / `lineSupplier` are untested — production-only composition that builds a live Bedrock client, installs an OS signal handler, and reads stdin**
- Why: the rubric's pinned tenet "tests are not optional" flags new logic without a test. `runInteractive` opens the session log, builds the `AgentLoop` via `AgentLoopFactory` (which resolves SigV4 credentials and constructs a real `BedrockRuntimeClient`), installs a `sun.misc.Signal("INT")` handler, and reads from `System.in`. Exercising it in a unit test would require a live AWS call (forbidden by the task), an OS signal, and an interactive stdin — none unit-reproducible. The *logic* it delegates to — the entire read-eval loop, every slash-command, the SIGINT→130 mapping + precedence, the inline-approval present-before-decide round-trip, and the turn-failure-resilience rules — IS tested via `ReplRunner` (100% line + branch) and `InteractiveApprover` (100% line + branch). This is a Minor (not a Blocker) because it mirrors the project's existing, documented treatment of the one-shot composition: `Main.class` is already JaCoCo-excluded (T-0.1) and `AgentLoopFactory.class` (T-0.9); the new interactive methods live inside the already-excluded `Main`, so the coverage gate honestly reflects "untestable composition root," and the bundle stayed above the 0.80 floor ("All coverage checks have been met").
- Suggestion: keep the exclude; the credentials-needed composition + the real Ctrl-C path are confirmed by the real-Bedrock smoke test at the G1 gate (the task names `codingagent` REPL as one of the two shapes G1 must exercise). No code change needed at T-1.1.

### Discussion

**[D1] [Discussion] 04-apis § 1.1 "multi-turn work" vs. AgentLoop.run starting a fresh transcript: T-1.1 drives each prompt as an independent loop turn; cross-turn model-context continuation is C15/T-1.2's replay→messages[] job**
- Why: 04-apis § 1.1 describes interactive mode as "the default; multi-turn work, approvals inline" — a *use* descriptor, not an AC that pins cross-turn context accumulation. `AgentLoop.run(String)` builds a *fresh* local transcript per call (it appends a `USER_MESSAGE` and seeds a new transcript), so two successive `ReplRunner` turns do not, by themselves, carry the prior turn's messages into the model's context. T-1.1's cited AC set (AC-10.1; 04-apis § 1.1/§ 1.4) does not pin cross-turn continuation, and the architecture explicitly assigns event→messages[] reconstruction to C15 / **T-1.2** ("Resume: replay events → messages[], latest-continuation default", refs AC-7.1/7.2/7.4, INV-1). The single-threaded-per-conversation invariant (02-architecture § 4) and the "REPL is a sequence of run(prompt) calls against a persistent loop+log+gate" framing in the dep-API note are both honoured: the REPL drives one turn at a time against one persistent `AgentLoop`/`EventLog`/`PermissionGate`, each prompt a new user turn appended to the same session log. A structural change to `AgentLoop` to thread a growing transcript across `run` calls would be a C2-owned change and is **not** required to satisfy T-1.1's ACs — so this is surfaced as a Discussion (not a `design-change-needed`), so a future reader sees the seam explicitly rather than mistaking the per-turn independence for a defect. `multiplePromptsRunSuccessiveTurns` documents the in-scope behaviour (a session accepts and runs multiple prompts); the cross-turn *memory* attaches at T-1.2.
- Suggested amendment kind: none (no spec change). The architecture already pins T-1.2 as the owner of replay→messages[]; when it lands, the REPL's `ReplLoop` seam can be fed a loop that continues the conversation (the seam is shaped for it — `ReplLoop` is the `AgentLoop::run` shape and can be swapped for a continued-conversation driver without touching `ReplRunner`). No code change needed at T-1.1.

### Praise

**[P1]** The REPL is split along the same seam that made the one-shot path testable without a "no live AWS" exception: all read-eval logic, every slash-command, the SIGINT→130 mapping + precedence, and the turn-failure-resilience rules live in `ReplRunner` (driven by injected seams — a `ReplLoop` functional seam fed a real `AgentLoop` over a scripted Bedrock double for the inline-approval test, and substitute loop seams for the branch tests), so the system under test (the loop orchestration) is never mocked, only Bedrock is. The SIGINT design reconciles the real handler with the existing seam cleanly: the non-portable `sun.misc.Signal` install is isolated in the coverage-excluded `Main` bootstrap, while the testable interrupt-to-130 logic in `ReplRunner` keys off both an injected `BooleanSupplier` flag (set by the real handler) **and** a caught `InterruptedRunException` (the seam OneShotRunner already maps) — so the precedence rule ("130 always wins") is enforced *structurally* by catch order, and `Main`'s handler interrupting the session thread is what makes the in-flight step cancellable (CT-SM-4). `InteractiveApprover` realizes AC-10.1 with a test that asserts the *ordering* contract, not just field presence: the answer source asserts, at the moment it is consulted, that the exact operation is **already** on the prompt stream — proving "present before executing." It correctly plugs into the *existing* `Approver` seam (T-0.7) as the deliberate REPL twin of `NonInteractiveApprover`, fails closed on EOF/blank/unrecognized (no fat-finger ever runs a side effect), and shares one input source with the REPL so the gate's mid-turn prompt reads from the same terminal — verified end-to-end in `inlineApprovalShowsExactOperation` (the denied `mvn -q clean verify` is presented exactly and the tool's handler never runs, INV-8/AC-10.2). The library/CLI output split is honoured: `ReplRunner`/`InteractiveApprover` write only to injected streams; only the `Main` bootstrap touches `System.in`/`System.out`/`System.err`.

4. **SIGINT handling** — `sun.misc.Signal` appears only in `Main`. 5. **Line input** — `Main.lineSupplier` uses `BufferedReader::readLine`; the other `readLine` hits (`ConfigLoader`, `ReadFileTool`) read *files*, an unrelated concern. No duplicate interactive-input reader.

- **cli-exit-codes.md (§ 1 codes, § 2 precedence, § 3 guarantees):** `0` on clean `/exit` / EOF (G3 "Interactive mode returns 0 on clean /exit"); `130` on Ctrl-C (G3), enforced first by catch order + the polled flag so it wins precedence (§ 2 "130 always wins", CT-EX-5); `1` on a fatal `PersistenceException` (AC-13.4).
