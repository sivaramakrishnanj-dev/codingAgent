### Minors

**[m1] [Reliability] `pom.xml` (shade execution) — Benign shade overlapping-resource warnings from dual SLF4J artifacts**
- Why: `mvn clean verify` emits `WARNING` lines — `slf4j-api` and `slf4j-simple` both ship `META-INF/versions/9/module-info` and `META-INF/LICENSE.txt`, and the uber-jar's `MANIFEST.MF` overlaps. The plugin's own guidance ("usually this is not harmful") applies, and the launch works correctly, so this is informational hygiene, not a correctness defect. As later tasks add dependencies (AWS SDK v2 in T-0.5+), unmanaged overlaps can compound. Reference: maven-shade-plugin overlapping-resources note.
- Suggestion: when the dependency set grows, consider an `AppendingTransformer`/`metaInf-services` strategy or a minimize-jar pass to silence the noise. No action required at T-0.1; the shaded jar is correct (verified manifest `Main-Class` + exit-0 launch).

### Nits
None.

### Praise

**[P1] [API-Design] `Main.run(String[])` seam vs. `Main.main(String[])`**
- The launch logic returns an `int` exit code from `run()`, and `main()` is a thin `System.exit(run(args))` shell. This keeps `System.exit` out of the unit-test path (so tests cannot terminate the JVM) while leaving the natural extension seam for the full exit-code contract (T-0.9). Effective Java Item 4 (noninstantiable utility via private constructor) is also observed. A deliberately testable scaffold.
