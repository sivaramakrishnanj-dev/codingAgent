## Summary

D11 implements the DCR-2 (ADR-0012 amended 2026-06-23) amendment plus the bundled D1 Converse-wiring
fix. Three things land:

1. **Multi-turn phase dialogue with in-phase transcript carry.** `GreenfieldDriver.run` no longer runs
   each pre-approval phase as a single model turn. Each pre-approval phase (requirements/design/tasks)
   now runs as a multi-turn conversation in `runPreApprovalPhase`: the phase opens automatically (first
   round driven by the phase framing + the approved upstream artifacts, plus the session's opening
   developer input for requirements), and each round the driver records the round into an in-phase
   transcript and **injects that transcript into the next round's prompt** (`phaseTurnPrompt`), so the
   model sees its own prior turns within the phase (AC-1.1, the fix for the in-phase discontinuity).
2. **Approve = finalize.** Each round the driver writes the latest deliverable to the phase artifact in
   code then consults the existing `ArtifactApprovalGate`. On **approval** the gate stamps the converged
   artifact (AC-1.5) and (at the tasks gate) verifies AC-2.5 traceability over the driver-written
   breakdown, and the driver advances. A **non-approve** (any reason — developer decline or a refused
   approval) does NOT persist-and-stop: the driver takes another refining turn from the new
   `DeveloperTurnSource` seam (AC-2.4); only when the developer supplies no further turn (end-of-input)
   does the phase pause `AWAITING_APPROVAL` (resumable, no source written — AC-1.4). The capture-and-
   persist trigger thus moves from each phase's first `END_TURN` (DCR-1) to phase approval, so the
   *converged* deliverable is what is finalized.
3. **D1 output-token cap.** `ConverseWireMapper` gains an optional `maxOutputTokens` (constructor
   overload + `GREENFIELD_MAX_OUTPUT_TOKENS = 16384`); when set it puts `inferenceConfig.maxTokens` on
   the request. `ModelClient.forGreenfield(bedrock)` builds a client with the 16K-capped mapper; the
   composer exposes it (`greenfieldModelClient()`) and `AgentLoopFactory` assembles each greenfield
   phase loop over it. The brownfield/one-shot/sub-agent path keeps the uncapped client (backend default
   cap, unchanged).

**Preserved (no regression):** AC-1.4 source-tool withholding across every pre-approval turn (the
phase-scoped registry is unchanged; `GreenfieldWiringTest` still asserts no source-change tool in a
pre-approval prompt), DCR-1 driver-authored persistence (`GreenfieldArtifactStore.write()`), cross-phase
artifact injection (`advancePrompt`), AC-2.5 tasks-gate traceability over the driver-written artifact,
AC-1.5 stamp, AC-2.3 per-phase approval before implementation.

**The load-bearing new tests.** `GreenfieldDriverTest.nonApproveAnswerDrivesAnotherRefiningTurn` (a
non-approve drives a second round, not a stop — AC-2.4); `refiningRoundPromptCarriesPriorInPhaseTurn`
+ `laterPhaseRefiningRoundReplaysPriorTurnWithNoOpeningDeveloperLine` (in-phase transcript carry —
AC-1.1); `approveFinalizesTheConvergedDeliverable` / `GreenfieldArtifactAuthoringTest
.approvalPersistsTheConvergedRequirementsNotTheFirstDraft` (the converged, not first-round, deliverable
is finalized — AC-1.5/DCR-2); `GreenfieldWiringTest.greenfieldPhaseRequestCarriesOutputBudgetOnTheWire`
+ `ConverseWireMapperTest$OutputBudget` (the greenfield request carries `inferenceConfig.maxTokens =
16384` — D1, with no live Bedrock).

### Discussion

**[D1] [Discussion] ADR-0012 / AC-1.5 say the driver persists "on approval"; the implementation writes the latest deliverable each round *before* the gate (a truncating overwrite finalized at approval).**
- Spec citation: ADR-0012 ("On phase approval ... the driver captures the model's latest substantive
  deliverable text in the phase conversation and writes it to the phase artifact ... the
  `ArtifactApprovalGate` then appends the AC-1.5 approval stamp"); AC-1.5 ("on approval the driver
  captures the converged deliverable ... persists it").
- Observation: the implementation writes the round's deliverable to the artifact *before* consulting
  the gate each round, not only at approval. This is required by two existing, unchanged collaborators:
  (a) `InteractiveGreenfieldApproval.approve()` *presents* the deliverable by reading the artifact from
  the store before prompting the developer, and (b) `ArtifactApprovalGate.enforceTraceability` reads the
  written tasks artifact to verify AC-2.5 — both need the artifact written before the gate runs. The
  observable contract the amended spec pins still holds: a non-approve never *finalizes* (no stamp, the
  conversation continues, the next round's truncating write supersedes), and the artifact that is
  stamped-and-kept is the converged one present at approval (asserted by
  `approvalPersistsTheConvergedRequirementsNotTheFirstDraft`). The wording "on approval" reads as
  "persist-and-stamp at approval"; the code's per-round write-before-present is the mechanism that
  realizes it given the present-then-decide gate.
- Suggested amendment kind: `ac-update` (a one-line clarification in AC-1.5 / ADR-0012 that the artifact
  is written each round to back the present-before-confirm gate, and *finalized* — stamped, advanced —
  at approval). Not load-bearing; does not block. Surfaced per the task's "spec-prose-vs-code wording
  observations as Discussion findings (do not block)".

## Reuse / duplication
Reuse-swept the new surface:
- **Output-token cap.** `inferenceConfig` / `maxTokens` appears at exactly one production site
  (`ConverseWireMapper`); no pre-existing inference-config setter exists to reuse. The cap is carried as
  optional mapper state (the lowest-blast-radius greenfield-scoped threading) rather than widening
  `AgentLoop`/`ModelClient.converse` signatures across the brownfield path — `ModelClient.forGreenfield`
  is a thin factory mirroring the existing `ModelClient(bedrock)` shape.
- **`DeveloperTurnSource`.** No pre-existing developer-turn-source abstraction exists in `workflow/` or
  `loop/`. It is the phase-typed seam over the REPL's raw `Supplier<String>` line source (parallel to how
  `ArtifactApprovalGate.ApprovalDecision` wraps the same supplier), not a duplicate; the phase-typed
  `nextTurn(phase)` is intentional (self-documenting + per-phase scriptable in tests).
- **Composer `greenfieldModelClient()`** mirrors the existing `modelClient()` accessor exactly; the
  12-arg convenience constructor delegating to the 13-arg one avoids breaking ~11 existing composer
  construction sites. The per-test turn-source / writer doubles are per-scope scaffolding, not production
  duplication.
