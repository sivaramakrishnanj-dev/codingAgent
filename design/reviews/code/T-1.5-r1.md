## Summary

T-1.5 delivers the output-disposal half of the Context Manager (C6, US-19, ADR-0006) as a standalone, injectable, fully unit-tested unit in the brand-new `com.srk.codingagent.context` package, and wires it into the agent loop's tool-result boundary (a minimal, in-architecture C2 edit). Three new types: `OutputDisposer` (head+tail truncation over the configured inline cap, with a `truncated` marker and a `fullRef`), `OutputRetrieval` (the AC-19.3 retrieve-from-log path over `SessionStore`), and `FullRef` (the `evt:<seq>` pointer format pinned in one place so writer and reader cannot drift). The disposal split: the loop persists the **full** output to the event log first (the log is the durable full store, AC-19.2; `append` returns the stamped `seq`), then reduces what enters the model context (AC-19.1) — head+tail with the tail kept because build/test failures are legible from the tail (ADR-0006). The reduced content is always plain text, the wire-valid Converse `toolResult.content.text` shape (D2-safe). Tier (3) summarize-via-model-call is correctly NOT built (the explicit escalation, OOS). The cap is `config.outputMaxInlineBytes()` (default 16384, NFR-OUTPUT-MAX-INLINE), measured in UTF-8 bytes, never a literal. `mvn clean verify` is green (586 tests; JaCoCo gate met). No Blockers, no Majors, no Minors. One stated assumption surfaced (the persisted-event representation, Reading A); no Discussion items.

## C2 wiring edit (noted explicitly per the task directive)

The truncation math stays entirely in the tested `context` unit; the loop edit is the call only. AgentLoopTest stays green (24 methods, incl. 3 new disposal-wiring tests); the existing CT-INV-2 event-order test is unaffected (the small outputs those tests produce pass through un-reduced, and disposal changes the returned content block, not the TOOL_RESULT event count).

### Praise

**[P1] The D2-class disposal contract is asserted precisely, not as a naive `truncated==true`.** The exact-boundary pair (`exactlyAtCap_notReduced` returns the same block; `oneByteOverCap_reduced` changes it) pins AC-19.1's "exceeding" word on both sides of the cap. `reduction_keepsTail` asserts the literal error tail (`"BUILD FAILED at line 42"`) survives — the failure mode ADR-0006 exists to avoid, and the one a head-only truncator would silently regress while still passing a "marker present" test. `multiByteCodePoint_notSplit` proves byte-boundary slicing never emits a replacement char. This is the structurally-blind-test defense the task called out.

**[P2] AC-19.3 is realized as a real round-trip, not asserted as "the data is in the log."** `OutputRetrievalTest.retrieve_roundTripsFullOutput` persists a 40 KB output through a real `SessionStore`+`EventLog` on a temp dir, derives the `fullRef` from the appended seq (exactly as the loop does), and asserts retrieval returns the byte-for-byte original. `AgentLoopTest.oversizedOutputFullPersistedAndRetrievable` drives the **whole loop** over a disk-backed log with a >16 KB `run_command` output, extracts the `fullRef` the model actually received in the reduced content's marker, and resolves it back to the full output — the exact G1 explore→verify→retrieve cycle the live smoke test depends on.

## Reuse / duplication

- **No pre-existing truncation / head-tail / disposal helper to merge with.** Sweep across `src/main` for `truncat*` / `headTail` / `reduceForContext` / `disposal` returns only (a) the `truncated`/`fullRef` data fields on `ToolResultPayload` and `CommandResult` (the schema fields T-1.5 makes live — expected, not duplication) and (b) doc references in `CommandExecutor`/`CommandResult`/`ReadFileTool` deferring disposal to T-1.5. The `context` package is brand new; this is the first disposal logic in the codebase.
- **The `forConfig` idiom is mirrored, not duplicated.** `OutputDisposer.forConfig(config)` follows the established config-wired control-unit shape (`VerifyLoop.forConfig`, the `BudgetGuard.NONE`-style seam) — same idiom, distinct concern. The cap is drawn from `config.outputMaxInlineBytes()`, never re-pinned (no literal `16384` in the production class; the `16384` literals are confined to tests asserting the NFR default).
- **The `getBytes(UTF_8).length` byte-count idiom** appears in `EditFileTool`/`WriteFileTool` as a one-liner; not extracted into a shared util because a one-line idiom is not a duplicate-able utility (extracting it would be over-abstraction, no behavior shared).
- **The schema-validation harness is reused, not copied.** `DisposedEventSchemaContractTest` uses the same `networknt` validator + classpath schema mapping `EventSchemaContractTest` (CT-SCH-1) uses, applied to disposal events — the test-side equivalent of validating against the formal artifact, not the deserializer's incidental rejection.

## Design-doc alignment

- **AC-19.1** (output exceeding NFR-OUTPUT-MAX-INLINE is reduced, not inlined whole): `reduceForContext` returns the block unchanged at/under the cap and a head+tail reduction over it; the loop returns the reduced block into the transcript. Boundary asserted both sides (`exactlyAtCap`/`oneByteOverCap`) and end-to-end (`oversizedOutputReducedForContext` / `smallOutputNotReduced`). ✓
- **AC-19.2** (full output persisted to the log so it remains retrievable): the loop appends `ToolResultPayload.of(id, status, result.content())` — the **full** content — before reducing; the log is the full store. `DisposedEventSchemaContractTest.fullPersistedToolResult_validates` + `OutputRetrievalTest.retrieve_returnsFullNotReduced` assert the full output is stored, schema-valid, marker-free. ✓
- **AC-19.3** (retrieve the full output from the log rather than re-run): `OutputRetrieval.retrieve` resolves the `fullRef` to the TOOL_RESULT event at that `seq` and returns its full content; round-trip asserted at unit and loop level. Malformed / unresolved / non-tool-result references surface as `Optional.empty()`, not a crash. ✓
- **NFR-OUTPUT-MAX-INLINE = 16 KB, configurable** (cap is the config value, in bytes): `forConfig` uses `config.outputMaxInlineBytes()`; `forConfig_defaultCapIs16384` pins 16384/16385; `forConfig_usesConfiguredCap` proves a custom cap is the live threshold. Byte length is UTF-8 (the field name + "16 KB" wire reality). ✓
- **ADR-0006 § Output disposal** (tiers 1+2 + retrieval; head+tail default with a `truncated` marker; tail kept; summarize is the escalation, OOS): tier (1) full-persist, tier (2) head+tail with marker, and retrieval are built; `reduction_keepsTail` pins the tail; no model-call summarizer exists (sweep clean — `summarizerModelId` is read nowhere new). ✓
- **D2 family (wire-valid toolResult)**: the reduced content is always a `String`, which `ConverseWireMapper.toWireToolResult` routes to the `text` member (a `Map` would hit `json`); `reducedContent_isPlainText` asserts the shape. No reduction produces a content the next Converse call would reject. ✓
- **Component C6 boundary respected** — disposal logic lives entirely in `com.srk.codingagent.context`; the `AgentLoop` edit is the disposer *call* only (no truncation math in the loop). Compaction (the other C6 mechanism) is untouched (M2 / T-2.1/T-2.2). ✓
- **Scope discipline** — tier (3) summarize-via-model-call is left unbuilt (the explicit escalation); no live Bedrock call anywhere (disposal is pure string/log logic; `aws_consulted` is empty). ✓
