## Summary

This task resumes the in-flight greenfield resilience cluster after the DCR-3 amendment landed. The prior
round (a) FIXED **D13** (a new run silently truncate-overwriting a prior run's APPROVED artifact) in code
and (b) raised **D12** (no greenfield mid-flow resume) as a DCR because the resume contract was unpinned.
DCR-3 amended the spec (AC-7.6 new, AC-1.5 augmented, AC-7.3 brought forward, ADR-0012 greenfield-resume
clause, CT-GF-1/CT-GF-2) so D12 is now spec-backed. This round implements:

- **D12 — greenfield mid-flow resume (AC-7.6, ADR-0012).** A fresh `--mode greenfield` run reconstructs
  its phase-state from the target repo's on-disk **approval-stamped artifacts** and resumes at the first
  phase whose artifact is unstamped/absent, rather than always restarting at requirements. A transient
  mid-phase failure left the failed phase unstamped, so re-derivation re-enters that same phase
  (retry-in-place) — it falls out of the same stamp rule. The already-approved phases' artifacts are
  pre-seeded so the resume phase still sees the approved upstream (DCR-1 cross-phase continuity, kept).
- **AC-7.3 repo-keying brought forward (C15).** `Main.ONE_SHOT_LINEAGE = "one-shot"` (the fixed M0
  placeholder, the run-collision root cause) is replaced by the real AC-7.3 repo key — git remote URL
  when present, else normalized absolute path (ADR-0005, 03-data-model § 2.1) — via an injectable,
  fully-tested `RepoKeyResolver`, wired into the greenfield, brownfield, and REPL session lineages in the
  composition root.
- **D13 (shipped) kept as-is.** The clobber guard is untouched; its stamp-detection is now *shared* with
  the resume re-derivation (one durable on-disk fact serves both roles, AC-1.5/ADR-0012).

### Praise

**[P1] [Reuse] One durable on-disk fact, two readers — no duplicated stamp-detection.** The AC-1.5 stamp
serves both the D13 clobber guard and the D12 resume re-derivation through the *single* private
`hasApprovalStampLine`/`carriesApprovalStamp` detector; the new `isApprovalStamped` accessor routes the
resume probe through it rather than re-implementing the scan. This is exactly the ADR-0012/AC-1.5 "one
durable fact" intent, and `resumeSignalAndClobberGuardAgreeOnTheStamp` pins that the two readers agree on
the same artifact.

## Reuse / duplication
Active-grep swept the new public surface:
- **Resume phase re-derivation.** No pre-existing greenfield phase-state reconstruction exists.
  `SessionReplay` is the *brownfield* events→`messages[]` projection (AC-7.2) — a deliberately distinct
  lane (D12 does NOT extend it; that was the rejected Option B in ADR-0012). `GreenfieldPhaseState` is the
  only greenfield phase-state reconstruction; no duplicate.
- **Repo-key derivation.** Only `RepoKeyResolver` (logic) + `Main.gitRemoteUrl` (the production
  subprocess) derive a repo key from a git remote. `SessionStore` *consumes* a repo key (its directory
  layout) but never derives one — its Javadoc explicitly defers derivation to "a later session task",
  which this is. No duplicate.
- **Stamp-detection.** Single source (`hasApprovalStampLine`), shared by the D13 guard and the new
  `isApprovalStamped`. No duplicate (and the deliberate `APPROVAL_STAMP_MARKER` string copy vs the
  workflow's `ApprovalStamp.MARKER` is the pre-existing, drift-pinned avoid-circular-dependency choice,
  unchanged this round).

`reuse_self_check: passed`.

## Design-doc alignment
- **AC-7.6 (greenfield mid-flow resume):** served — `GreenfieldPhaseState.reconstruct` resumes at the
  first unstamped/absent phase (`approvedRequirementsResumesAtDesign`, …`AtTasks`, …`AtImplement`); an
  interrupted mid-phase (unstamped) re-enters (`approvedRequirementsUnstampedDesignReEntersDesign`,
  `interruptedMidDesignReEntersDesign`); the driver honours it (`GreenfieldDriverResumeTest`); end-to-end
  over disk (`GreenfieldResumeContractTest`, CT-GF-1). No `[Design-Gap]`.
- **AC-7.3 (repo-keying):** served — `RepoKeyResolver` derives git remote URL else normalized abs path;
  distinct projects → distinct keys (`distinctProjectsGetDistinctKeys`); replaces `ONE_SHOT_LINEAGE` in
  the composition root. No `[Design-Gap]`.
- **AC-1.5 (the stamp is the durable resume + clobber fact):** served — one detector, two readers
  (`resumeSignalAndClobberGuardAgreeOnTheStamp`).
- **ADR-0012 (accepted tradeoff — in-phase transcript not preserved):** respected — only phase boundaries
  are re-derived; on a later-phase resume the opening developer input is suppressed (the phase opens from
  its framing + pre-seeded approved upstream), and the in-phase transcript starts empty. No attempt to
  reconstruct in-phase turns (which would be the rejected Option B).
- **AC-7.1/7.2/7.4 (brownfield replay — out of scope):** respected — D12 did NOT extend `SessionReplay` /
  `SessionLineage` / `ResumeCommand`; greenfield phase-state is re-derived from the on-disk `design/`
  artifacts, the lanes kept distinct.
- **CT-GF-2 (no-clobber, D13):** already satisfied by the shipped D13 tests; confirmed still green and
  explicitly mapped (see handoff `contract_tests_satisfied`).
