## Summary

D6 wires the interactive REPL greenfield per-phase `ArtifactApprovalGate.ApprovalDecision` to the
developer's stdin: at each completed phase it presents that phase's deliverable (the
requirements/design/tasks artifact) and then reads a y/N from the same `answerSource`/stdin the
`InteractiveApprover` uses (AC-1.5 present-before-confirm; AC-10.1 present-before-decide precedent).
The defect — `interactiveGreenfield` passed `completedPhase -> false`, so a live greenfield REPL run
could never advance past requirements — is fixed by threading a new coverage-counted
`InteractiveGreenfieldApproval` into `AgentLoopFactory.createGreenfieldDriver`. The one-shot path stays
DECLINE-by-default (a non-interactive run cannot prompt). The affirmative-parsing is now a single
shared `AffirmativeAnswer` helper (no duplicate of the `y`/`yes` check). The change is surgical:
`GreenfieldDriver` and the existing CTs are untouched.

## Stated assumption (audit trail)

- **spec_ref:** AC-1.5 / AC-10.1 — *what "present the deliverable" reads from*
  - **reading:** The completed phase's deliverable presented before the y/N is the phase's authored
    design-doc artifact (`GreenfieldArtifact` → `design/0X-*.md`), read from the `GreenfieldArtifactStore`
    — since `ApprovalDecision.approve(GreenfieldPhase)` receives only the phase (not the `LoopOutcome`
    text), and the artifact is the durable, re-readable deliverable the model authored via
    `write_artifact`. This avoided changing `GreenfieldDriver`'s seam signature (keeping its CT green).
  - **alternative_considered:** Thread the phase loop's `LoopOutcome.finalText()` into the decision by
    widening the `ApprovalDecision`/`ApprovalGate` signature. Rejected: it would change the
    `GreenfieldDriver` contract the `GreenfieldDriverTest` pins (the task says the driver must not change
    unless genuinely necessary), and the artifact is the more faithful "deliverable" (the persisted
    requirements/design/tasks the developer is approving), readable as the spec's "requirements
    artifact" in AC-1.5.
